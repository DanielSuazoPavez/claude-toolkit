- **[TOOLKIT]** Declarative sync profiles — projects choose which resource categories to receive (`sync-profiles`)
    - **scope**: `toolkit`, `cli`
    - **notes**: Surfaced 2026-04-28 during web ecosystem exploration. Base dist currently syncs ~34 skills to every project regardless of relevance — schema-smith gets `design-docker`, `design-aws`, `brainstorm-idea`, etc. Each unused skill still appears in the session-start skills list, consuming context tokens for zero value. VILA-Lab's "context as scarce resource" principle and Crosley's migration story (400 lines of always-loaded rules → contextual skills → 40% context reduction) both validate: don't ship what won't be used. Start with a design pass to decide granularity — category-based is simpler but domain-based maps better to actual usage patterns. - **[HOOKS]** Bash output compression hook — token savings via smart output filtering (`bash-output-compression-hook`)
    - **scope**: `hooks`
    - **notes**: Surfaced 2026-04-28 during web ecosystem exploration (REPOZY/superpowers-optimized). PreToolUse hook intercepts Bash calls, classifies via regex, compresses output before it enters context. Two tiers: tier 1 (near-lossless: git add/commit/push/pull/clone/fetch, npm/pip/cargo install → one-line summaries) and tier 2 (smart filtering: git status hint lines removed, git log truncated at 30 lines, passing tests → summary only, successful builds → summary, lint output grouped by severity, large ls/find truncated at 50/60 entries). Safety rails: never compresses diffs, file reads (cat/head/tail), piped commands, --verbose/--debug, curl/wget, failed commands (non-zero exit), or output under 200 chars. Adaptive re-run detection: same command within 60s passes through raw (handles "compressed output was insufficient"). Fail-open: any hook error → original command runs unmodified. Architecture: PreToolUse rewrites compressible commands to run through an optimizer script that executes + compresses; non-compressible passes through unchanged. ~76% token savings claimed on mixed sessions. Full source reviewed — well-engineered, zero-deps Node.js. Tasks: (1) evaluate feasibility as a toolkit hook (bash vs Node.js implementation, interaction with existing grouped-bash-guard); (2) decide tier scope — which compressions are worth the complexity for our workflow; (3) prototype and measure actual token savings in real sessions. Exploration entry: `output/claude-toolkit/exploration/BACKLOG.md`.

- **[TOOLKIT]** Study targeted patterns from REPOZY/superpowers-optimized (`study-superpowers-optimized-patterns`)
    - **scope**: `toolkit`, `agents`, `docs`
    - **notes**: Surfaced 2026-04-28 during web ecosystem exploration. Production fork of obra/superpowers (already in curated resources) — 24 skills, 10 hooks, research-backed design. Not adopting the system, but three patterns warrant deeper study (bash compression split to its own task `bash-output-compression-hook`): (1) **Self-consistency reasoning** — embedded technique (not standalone) in debugging + verification skills. Generates N independent reasoning paths (3-7 depending on complexity), majority vote, acts based on agreement level (≤50% → stop and report ambiguity). Based on Wang et al. ICLR 2023. Evaluate whether this pattern improves our `code-debugger` and `goal-verifier` agents. (2) **AGENTbench findings on instruction volume** — paper "Evaluating AGENTS.md" (138 tasks) found LLM-generated context files decreased success by 2-3% while increasing costs 20%. Explore implications for how we generate and evaluate toolkit resources. (3) **"Do LLMs Benefit from Their Own Words?"** — removing prior assistant responses maintained quality while reducing context 5-10x. Models over-condition on their own output. Explore implications for session continuity and subagent prompt design. (4) **"Compliance ≠ competence" as resource-generation lens** — the finding that more instructions can actively hurt is worth evaluating as a heuristic/checklist item in `create-skill`, `create-agent`, `create-docs`, and `evaluate-*` skills: "will adding this instruction actually improve outcomes, or just increase context load?" Exploration entry: `output/claude-toolkit/exploration/BACKLOG.md`.
