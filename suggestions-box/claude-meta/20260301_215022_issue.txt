Consider splitting 'command-like' skills into a separate resource type with its own evaluation criteria. Current evaluate-skill rubric scores on 'knowledge delta' (D1) which doesn't fit procedural/trigger skills like snap-back, wrap-up, write-handoff. These are 'commands with extra steps' — their value is consistency and completeness, not expert knowledge. Options: (1) Add a 'type' field to skill frontmatter and branch evaluation, (2) Revive commands/ as a separate resource type with its own lighter evaluator (evaluate-command), or (3) Keep everything in skills/ but create a second rubric dispatched by type. A command evaluator would focus on: trigger correctness, step completeness, output format, anti-patterns — maybe 4 dimensions, ~60 points, no knowledge delta question.
